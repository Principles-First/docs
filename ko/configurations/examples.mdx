## 개요

이 페이지는 다양한 사용 사례에 대한 완전한 복사-붙여넣기 가능한 설정 예시를 제공합니다. 각 예시에는 SDK 초기화와 관련 런타임 설정이 모두 포함되어 있습니다.

---

## 최소 설정

가장 간단한 시작 설정:

```javascript
window.Moss.init({
    applicationId: 'YOUR_APP_ID',
    userId: 'USER_ID'
})
```

모든 대시보드 기본값을 사용합니다. 빠른 테스트나 대시보드 설정이 요구 사항을 충족하는 경우에 적합합니다.

---

## 파워 유저 설정

포괄적인 지원이 필요한 사용자를 위한 전체 기능 설정:

```javascript
window.Moss.init({
    applicationId: 'YOUR_APP_ID',
    userId: 'premium-user-123',

    // 모든 기능 활성화
    useVision: true,
    enableSessionRecording: true,
    enableScreenHistoryRecording: true,
    language: 'ko',

    // 고급 플래닝
    planning: {
        enabled: true,
        maxSteps: 15,
        allowUserSkip: true,
        showPlanOverview: true
    },

    // 프리미엄 사용자를 위한 확장된 세션
    sessionConfig: {
        inactivityTimeout: 60,
        maxSessionAge: 4,
        showNotification: true
    }
})
```

## 개발자 / 디버그 설정

개발 및 디버깅에 최적화된 설정:

```javascript
window.Moss.init({
    applicationId: 'YOUR_APP_ID',
    userId: 'dev-user',

    // 디버그 설정
    debugMode: true,
    logLevel: 'DEBUG',

    // 빠른 테스트를 위한 최소 안정성 검사
    stability: {
        layers: {
            networkIdle: false,
            browserIdle: false
        }
    },

    // 더 빠른 테스트를 위한 뷰포트 스크린샷
    screenshotMode: 'viewport'
})
```

<Tip>
    더 나은 성능을 위해 프로덕션에서는 디버그 설정을 비활성화하는 것을
    잊지 마세요.
</Tip>

## 개인정보 중심 설정

개인정보에 민감한 사용자를 위한 최소 데이터 수집:

```javascript
window.Moss.init({
    applicationId: 'YOUR_APP_ID',
    userId: 'privacy-user-456',

    // 최소 데이터 수집
    enableSessionRecording: false,
    enableScreenHistoryRecording: false,
    useVision: true,  // 여전히 유용하지만 저장되는 데이터 감소

    // 짧은 세션
    sessionConfig: {
        inactivityTimeout: 15,
        maxSessionAge: 1
    }
})
```

### 런타임 개인정보 토글 포함

```javascript
// 녹화가 활성화된 초기 설정
window.Moss.init({
    applicationId: 'YOUR_APP_ID',
    userId: 'USER_ID',
    enableSessionRecording: true,
    enableScreenHistoryRecording: true
})

// 개인정보 토글 핸들러
function setPrivacyMode(enabled) {
    window.Moss.configure({
        enableSessionRecording: !enabled,
        enableScreenHistoryRecording: !enabled
    })
}
```

## 다중 등급 구독 설정

구독 수준에 따른 다양한 설정:

```javascript
const subscriptionConfigs = {
    free: {
        planning: { enabled: false },
        sessionConfig: {
            inactivityTimeout: 15,
            maxSessionAge: 1
        }
    },
    pro: {
        planning: {
            enabled: true,
            maxSteps: 10,
            allowUserSkip: true,
            showPlanOverview: true
        },
        sessionConfig: {
            inactivityTimeout: 30,
            maxSessionAge: 2
        }
    },
    enterprise: {
        planning: {
            enabled: true,
            maxSteps: 20,
            allowUserSkip: true,
            showPlanOverview: true
        },
        sessionConfig: {
            inactivityTimeout: 60,
            maxSessionAge: 8
        }
    }
}

function initMossForUser(user) {
    const tierConfig = subscriptionConfigs[user.tier] || subscriptionConfigs.free

    window.Moss.init({
        applicationId: 'YOUR_APP_ID',
        userId: user.id,
        ...tierConfig
    })
}
```

## 싱글 페이지 애플리케이션 (SPA)

React, Vue 또는 Angular 애플리케이션을 위한 설정:

<Tabs>
  <Tab title="React">
    ```tsx
    import { useEffect } from 'react'
    import { useAuth } from './auth'  // 인증 훅

    function App() {
        const { user, isAuthenticated } = useAuth()

        useEffect(() => {
            if (isAuthenticated && user) {
                window.Moss.init({
                    applicationId: process.env.REACT_APP_MOSS_APP_ID,
                    userId: user.id,
                    language: user.preferredLanguage,
                    planning: {
                        enabled: user.subscription === 'pro'
                    }
                })
            }
        }, [isAuthenticated, user])

        return <div>Your App</div>
    }
    ```
  </Tab>
  <Tab title="Vue">
    ```vue
    <script setup>
    import { watch } from 'vue'
    import { useAuth } from './composables/auth'

    const { user, isAuthenticated } = useAuth()

    watch([isAuthenticated, user], ([auth, u]) => {
        if (auth && u) {
            window.Moss.init({
                applicationId: import.meta.env.VITE_MOSS_APP_ID,
                userId: u.id,
                language: u.preferredLanguage,
                planning: {
                    enabled: u.subscription === 'pro'
                }
            })
        }
    }, { immediate: true })
    </script>
    ```
  </Tab>
  <Tab title="Angular">
    ```typescript
    import { Component, OnInit } from '@angular/core'
    import { AuthService } from './auth.service'

    @Component({
        selector: 'app-root',
        template: '<router-outlet></router-outlet>'
    })
    export class AppComponent implements OnInit {
        constructor(private auth: AuthService) {}

        ngOnInit() {
            this.auth.user$.subscribe(user => {
                if (user) {
                    (window as any).Moss.init({
                        applicationId: environment.mossAppId,
                        userId: user.id,
                        language: user.preferredLanguage,
                        planning: {
                            enabled: user.subscription === 'pro'
                        }
                    })
                }
            })
        }
    }
    ```
  </Tab>

## 고성능 설정

최소 오버헤드에 최적화:

```javascript
window.Moss.init({
    applicationId: 'YOUR_APP_ID',
    userId: 'USER_ID',

    // 스크린샷 처리 감소
    screenshotMode: 'viewport',

    // 최소 로깅
    logLevel: 'ERROR',
    debugMode: false,

    // 빠른 안정성 검사
    stability: {
        maxTotalWaitMs: 5000,
        layers: {
            networkIdle: true,
            domMutations: true,
            loadingIndicators: false,  // 로딩 인디케이터 감지 건너뛰기
            layoutShift: false,        // 레이아웃 시프트 감지 건너뛰기
            browserIdle: false,        // requestIdleCallback 건너뛰기
            finalFrame: true
        }
    }
})
```

## 모범 사례 요약

### 대시보드 vs SDK 설정

**대시보드 설정 사용 시기:**
- 애플리케이션 전체 기본값 설정
- 보안 정책 구성 (허용 도메인, API 키)
- 모든 사용자에게 적용되는 AI 지침 관리
- 거의 변경되지 않는 설정

**SDK 초기화 사용 시기:**
- 사용자별 사용자 정의가 필요한 경우
- 환경에 따른 설정 (개발/스테이징/프로덕션)
- 다른 사용자 등급에 다른 설정이 필요한 경우
- 기존 사용자 기본 설정과 통합

**런타임 설정 사용 시기:**
- 사용자 작업에 대한 응답 (개인정보 토글)
- 기능 플래그 또는 A/B 테스트 구현
- 세션별 재정의
- 조건에 따른 동적 조정

### 성능 팁

1. **스크린샷 모드**: 긴 페이지에는 `viewport` 사용
2. **안정성 레이어**: 필요하지 않은 경우 `loadingIndicators` 및 `layoutShift` 비활성화
3. **세션 녹화**: 개인정보에 민감한 페이지에서는 비활성화
4. **로그 레벨**: 프로덕션에서는 `WARN` 또는 `ERROR` 사용

### 보안 팁

1. 클라이언트 측 코드에 API 키를 노출하지 마세요
2. 대시보드에서 도메인 제한 사용
3. 세션 녹화에 대한 사용자 동의 구현
4. 해시된/익명화된 사용자 ID 사용
