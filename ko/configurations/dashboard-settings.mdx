## 개요

[Moss 대시보드](https://dashboard.viamoss.ai)는 애플리케이션 수준 설정을 관리하기 위한 웹 인터페이스를 제공합니다. 이러한 설정은 SDK 초기화 또는 런타임 설정으로 재정의할 수 있는 기본값 역할을 합니다.

<Tip>
    대시보드 설정은 애플리케이션 전체 기본값, 보안 정책 및 모든 사용자에게
    적용되어야 하는 설정에 이상적입니다.
</Tip>

---

## 애플리케이션 정보

애플리케이션의 메타데이터와 ID 설정을 보고 관리합니다.

### 애플리케이션 이름

대시보드에서 표시되는 애플리케이션 이름입니다. 이 필드는 현재 읽기 전용입니다.

### 애플리케이션 ID

고유한 애플리케이션 식별자입니다. SDK를 통합할 때 이를 사용합니다.

### 생성 날짜

애플리케이션이 생성된 시점입니다.

### UI 언어

SDK 인터페이스에서 최종 사용자에게 표시되는 언어를 제어합니다.

**기본값:** 자동 (사용자의 브라우저 언어 감지)

**설정 > 애플리케이션 정보**로 이동하여 언어 드롭다운에서 선택합니다.

<Info>
    "자동"으로 설정하면 Moss가 자동으로 사용자가 선호하는 브라우저 언어를
    감지하여 사용합니다. SDK 초기화를 통해 재정의할 수 있습니다.
</Info>

## 일반 설정

애플리케이션의 핵심 AI 어시스턴트 기능을 구성합니다.

### 비전 활성화

SDK가 스크린샷과 시각적 콘텐츠를 처리하고 분석할 수 있도록 합니다. 활성화되면 AI 어시스턴트가 사용자의 화면을 "보고" 더 상황에 맞는 가이드를 제공할 수 있습니다.

**기본값:** `true`

**설정 > 일반 설정 > 핵심 기능**으로 이동하여 "비전 활성화"를 토글합니다.

**사용 사례:**
- 시각적 문제 해결
- UI 요소 식별
- 스크린샷 기반 가이드

<Tip>
    사용자가 특정 UI 요소를 찾는 데 도움이 필요하거나 AI가 시각적 컨텍스트를
    제공하기를 원하는 경우 이 기능을 활성화하세요.
</Tip>

### 세션 리플레이 활성화

대시보드에서 리플레이할 수 있도록 사용자 세션을 녹화하고 업로드합니다. 사용자 상호작용을 검토하고 문제를 디버그할 수 있습니다.

**기본값:** `true`

**설정 > 일반 설정 > 핵심 기능**으로 이동하여 "세션 리플레이 활성화"를 토글합니다.

**녹화되는 내용:**
- 사용자 상호작용 (클릭, 입력, 내비게이션)
- DOM 구조 변경
- 타임스탬프 및 세션 메타데이터

<Warning>
    세션 녹화에는 민감한 사용자 데이터가 포함될 수 있습니다. 적절한
    개인정보 보호 정책과 사용자 동의를 확보하세요.
</Warning>

### AI용 화면 히스토리 활성화

AI의 컨텍스트에 사용자 액션 히스토리(클릭, 입력, 내비게이션)와 DOM 변경 사항을 포함합니다. AI가 사용자의 여정을 더 풍부하게 이해할 수 있습니다.

**기본값:** `true`

**설정 > 일반 설정 > 핵심 기능**으로 이동하여 "AI용 화면 히스토리 활성화"를 토글합니다.

**이점:**
- 더 상황에 맞는 AI 응답
- 사용자 의도에 대한 더 나은 이해
- 개선된 다단계 가이드

### AI 어시스턴트 지침

AI 어시스턴트가 애플리케이션을 탐색하거나 문서를 통해 얻을 수 없는 컨텍스트를 제공하는 데 도움이 되는 사용자 정의 지침을 제공합니다.

**기본값:** 비어 있음
**최대:** 30,000자

<Note>
    이 설정은 **대시보드 및 API 전용**입니다 - SDK 초기화를 통해 구성할 수
    없습니다.
</Note>

**설정 > 일반 설정 > AI 어시스턴트 지침**으로 이동하여 텍스트 영역에 사용자 정의 지침을 입력합니다.

**포함할 내용:**
- 애플리케이션 고유의 특정 내비게이션 패턴
- 중요한 UI 컨텍스트 또는 워크플로우 정보
- 특수 고려 사항 또는 엣지 케이스
- 도메인 고유의 용어 또는 개념

**예시:**

```
사용자가 '대시보드'에 대해 물으면 항상 개요가 아닌 분석 섹션으로
먼저 안내하세요. 메인 내비게이션 메뉴는 모바일 기기에서 접히며
왼쪽 상단의 햄버거 아이콘을 클릭해야 합니다.
```

<Tip>
    명확하고 간결한 언어를 사용하세요. AI는 사용자를 도울 때 이러한 지침을
    통합합니다.
</Tip>

## 사용자 경험

플래닝 기능과 세션 동작을 구성합니다.

### 플래닝 설정

복잡한 사용자 작업을 위한 AI 기반 다단계 플래닝을 활성화합니다.

#### 플래닝 활성화

플래닝 기능을 완전히 켜거나 끕니다.

**기본값:** `false`

**설정 > 사용자 경험 > 플래닝 설정**으로 이동하여 "플래닝 활성화"를 토글합니다.

활성화되면 AI가 복잡한 작업을 여러 단계로 분해하고 순차적으로 실행할 수 있습니다.

#### 최대 단계

단일 플랜에서 허용되는 최대 단계 수입니다.

**기본값:** `10`
**범위:** 1-20

<Info>
    높은 값은 더 복잡한 다단계 작업을 허용하지만 완료하는 데 더 오래
    걸릴 수 있습니다.
</Info>

#### 사용자 건너뛰기 허용

사용자가 플랜의 개별 단계를 건너뛸 수 있도록 합니다.

**기본값:** 활성화

활성화되면 사용자가 이미 완료했거나 수동으로 처리하고 싶은 단계를 건너뛸 수 있습니다.

#### 플랜 개요 표시

실행 전에 사용자에게 전체 플랜 개요를 표시합니다.

**기본값:** 활성화

AI가 작업을 수행하기 전에 사용자에게 무엇을 할 계획인지 보여주어 투명성과 신뢰를 향상시킵니다.

### 세션 동작

사용자 세션이 관리되고 타임아웃되는 방식을 구성합니다.

#### 비활성 타임아웃

비활성 세션이 만료된 것으로 간주되기 전의 시간(분)입니다.

**기본값:** 30분

이 비활성 기간 후에 사용자를 위한 새 세션이 생성됩니다.

#### 최대 세션 기간

활동에 관계없이 최대 세션 지속 시간(시간)입니다.

**기본값:** 2시간

사용자가 활성 상태여도 이 기간 후에 세션이 자동으로 만료됩니다.

#### 알림 표시

새 세션을 시작할 때 알림을 표시합니다.

**기본값:** 활성화

활성화되면 새 세션이 시작될 때 사용자에게 알림이 표시됩니다.

## 보안

SDK의 접근 및 보안 설정을 제어합니다.

### 도메인 접근 제어

무단 사용을 방지하기 위해 SDK를 사용할 수 있는 도메인을 제한합니다.

**기본값:** 비어 있음 (모든 도메인 허용)

<Note>이 설정은 **대시보드 전용**입니다.</Note>

**작동 방식:**
- 목록이 비어 있으면 SDK가 모든 도메인에서 작동합니다
- 도메인이 지정되면 SDK가 해당 도메인에서만 작동합니다
- 와일드카드는 지원되지 않습니다

**허용 도메인 예시:**

```
app.example.com
staging.example.com
localhost:3000
```

<Warning>
    도메인을 제한할 때 주의하세요. 애플리케이션이 실행되는 모든 환경
    (프로덕션, 스테이징, 개발)을 포함해야 합니다.
</Warning>

## 개발자 옵션

개발자 및 기술 설정을 위한 고급 설정입니다.

### API 키

SDK 인증을 위한 API 키를 관리합니다.

<Note>이 설정은 **대시보드 전용**입니다.</Note>

#### 현재 API 키

활성 프로덕션 API 키입니다. 전체 키는 생성 시 한 번만 표시됩니다.

<Warning>
    API 키는 민감합니다. 안전하게 저장하고 절대 버전 관리에 커밋하지
    마세요.
</Warning>

#### API 키 재생성

새 API 키를 생성하고 이전 키를 무효화합니다.

<Danger>
    API 키를 재생성하면 즉시 이전 키가 무효화됩니다. 이전 키를 사용하는
    모든 애플리케이션이 작동을 중단합니다.
</Danger>

### 벡터 저장소 설정

RAG(검색 증강 생성)에 사용할 문서 시스템을 선택합니다.

<Note>이 설정은 **대시보드 전용**입니다.</Note>

**기본값:** V1 시스템 (새 애플리케이션용)

**옵션:**
- **레거시 시스템:** documentation_records 테이블 사용
- **V1 시스템 (신규):** 3계층 문서 시스템 사용 (소스, 페이지, 청크)

<Info>
    V1은 소스, 페이지, 청크로 더 나은 구성을 제공합니다. 또한 전체 페이지
    콘텐츠를 보존합니다.
</Info>

**전환 전:**
- 문서가 새 시스템에 수집되었는지 확인
- 전환 후 RAG 검색 결과 테스트
- 필요시 언제든지 다시 전환 가능

### 위험 영역

애플리케이션에 영향을 미치는 되돌릴 수 없는 작업입니다.

<Note>이 설정은 **대시보드 전용**입니다.</Note>

#### 애플리케이션 삭제

애플리케이션과 모든 관련 데이터를 영구 삭제합니다.

**상태:** 곧 출시 예정

<Danger>
    활성화되면 이 작업은 세션, 사용자 및 지식 기반 문서를 포함한 모든
    데이터를 영구적으로 제거합니다. 취소할 수 없습니다.
</Danger>
