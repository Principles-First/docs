## 소개

Moss SDK는 애플리케이션의 요구에 맞게 다양한 방법으로 구성할 수 있습니다. 이 유연성을 통해 합리적인 기본값을 설정하면서 개발자에게 사용자 또는 세션별로 세밀한 제어를 제공할 수 있습니다.

### 설정 방법

| 방법 | 사용 사례 | 우선순위 |
|------|----------|----------|
| **대시보드 설정** | 애플리케이션 전체 기본값, 보안 정책 | 최저 |
| **SDK 설정** | 배포별 사용자 정의, 환경별 설정 | 최고 |

## 설정 우선순위

동일한 설정이 여러 곳에서 구성된 경우, Moss는 다음 우선순위를 사용합니다 (높은 것부터 낮은 순):

1. **SDK 설정** - `AgentProvider` config 옵션
2. **대시보드 설정** - 애플리케이션 수준 기본값

<Info>
    이 우선순위 시스템을 통해 대시보드에서 합리적인 기본값을 설정하면서
    개발자가 코드에서 특정 설정을 재정의할 수 있습니다.
</Info>

### 예시: 우선순위 동작

```tsx
// 대시보드 설정: language = 'en'
// SDK 설정이 이를 재정의:
<AgentProvider config={{
  apiUrl: 'https://clippy-api.viamoss.ai',
  applicationId: 'YOUR_APP_ID',
  userId: currentUser.id,
  getJwt: () => fetchMossToken(),
  language: 'ko',  // 대시보드보다 우선
}}>
  <AssistantButton />
</AgentProvider>
```

## 빠른 참조

### 대시보드 전용 설정

이 설정은 대시보드를 통해서만 구성할 수 있습니다:

- AI 어시스턴트 지침
- API 키
- 도메인 접근 제어
- 벡터 저장소 설정
- 애플리케이션 삭제

### SDK로 구성 가능한 설정

이 설정은 대시보드에서 설정하고 SDK를 통해 재정의할 수 있습니다:

| 설정 | 대시보드 | SDK 설정 |
|------|----------|----------|
| 언어 | 예 | 예 |
| 비전 활성화 | 예 | 예 |
| 세션 녹화 | 예 | 예 |
| 화면 히스토리 | 예 | 예 |
| 플래닝 | 예 | 예 |
| 세션 타임아웃 | 예 | 예 |
| 스크린샷 모드 | 예 | 예 |
| 로그 레벨 | 예 | 예 |
| 디버그 모드 | 예 | 예 |
| DOM 안정성 | 예 | 예 |
