# Forced Workflow

Forced Workflow는 웹페이지 사용자에게 사전 준비된 Moss 가이드를 자동으로 시작합니다.  
사용자가 챗봇과 상호작용하기 전에 선제적으로 가이드를 제공하고자 할 때 적합한 기능입니다.

<Warning>
  Forced Workflow는 현재 Beta 기능입니다. Production에 적용하기 전에 Moss와
  flow 구성을 검토하세요.
</Warning>

<Info>
  `activationQuery`, 페이지 매칭 조건, 문구는 Moss 대시보드에 설정된
  가이던스 플로우와 일치해야 합니다.
</Info>

일반적인 SDK 설치에는 이 기능을 사용하지 않습니다.

## 시작 전 준비

다음이 필요합니다:

- `apiUrl`, `applicationId`, `userId`, `getJwt`가 포함된 SDK 설치
- 각 `activationQuery`에 연결된 Moss 가이던스 플로우
- Flow를 보여줄 사용자를 결정하는 사용자 또는 실험 플래그
- Flow를 보여줄 위치를 결정하는 route 또는 DOM 조건

## 기본 설정

기존 `AgentProvider` 설정에 `displayMode`, `headless`, 선택적
`stability.afterReload`, `forcedActivationFlow`를 추가합니다.

```tsx App.tsx
import { AgentProvider, AssistantButton } from "@viamoss/moss-sdk";

function shouldStartWorkflow({ url }: { url: URL; document: Document }) {
  return url.pathname === "/";
}

function App() {
  return (
    <>
      <YourApp />
      <AgentProvider
        config={{
          apiUrl: "https://clippy-api.viamoss.ai",
          applicationId: "YOUR_APP_ID",
          userId: currentUser.id,
          getJwt: () => fetchMossToken(),

          displayMode: "headless",
          headless: {
            spotlight: true,
            animatedCursor: true,
            completionToast: {
              enabled: true,
              message: "Workflow complete.",
            },
          },
          stability: {
            afterReload: {
              enabled: true,
              pendingTTL: 60000,
            },
          },
          forcedActivationFlow: {
            enabled: currentUser.isNewUser,
            triggerOn: "first_visit",
            delayMs: 500,
            flows: [
              {
                id: "FLOW_ID",
                activationQuery: "ACTIVATION_QUERY",
                shouldTrigger: shouldStartWorkflow,
                showAskButton: false,
                title: "Welcome",
                description: "Let us show you the key steps.",
                acceptText: "Show me",
                skipText: "I'll explore on my own",
              },
            ],
          },
        }}
      >
        <AssistantButton />
      </AgentProvider>
    </>
  );
}
```

## 여러 Flow

여러 가이던스 플로우를 동시에 운영하는 것도 가능합니다. 단, 이 경우 사용자 그룹과 노출 조건을 명확히 나누어 관리해야 합니다.
나중에 보여줄 flow는 `requiresCompleted`로 이전 flow가 완료되었다는 조건이 만족된 경우에만 표시할 수 있습니다. 특정 페이지에서 시작해야 하는 flow에는 `startUrl`을 사용하세요.

```tsx
forcedActivationFlow: {
  enabled: currentUser.isNewUser,
  triggerOn: 'first_visit',
  flows: [
    {
      id: 'intro',
      activationQuery: 'INTRO_GUIDE_QUERY',
      shouldTrigger: isHomePage,
      showAskButton: false,
      title: 'Welcome',
      description: 'Start with the main page.',
      acceptText: 'Show me',
      skipText: "I'll explore on my own",
    },
    {
      id: 'upload',
      activationQuery: 'UPLOAD_GUIDE_QUERY',
      requiresCompleted: ['intro'],
      shouldTrigger: isUploadPage,
      startUrl: '/upload',
      showAskButton: true,
      title: 'Upload your first file',
      description: 'Follow these steps to upload content.',
      acceptText: "Let's upload",
      skipText: 'Maybe later',
    },
  ],
}
```

## 옵션

| 옵션                           | 설명                                                                                         |
| ------------------------------ | -------------------------------------------------------------------------------------------- |
| `displayMode: 'headless'`      | 채팅창 대신 페이지 위의 작은 안내 UI로 가이드를 표시                                         |
| `headless.spotlight`           | 사용자가 봐야 할 영역을 더 쉽게 찾을 수 있도록 강조                                          |
| `headless.animatedCursor`      | 다음으로 이동할 위치를 커서 애니메이션으로 안내                                              |
| `stability.afterReload`        | 페이지 새로고침 후에도 진행 중인 안내를 이어감                                               |
| `forcedActivationFlow.enabled` | Forced Workflow 활성화 여부                                                                  |
| `triggerOn`                    | 가이드를 언제 다시 보여줄지 설정 (첫 방문, 세션당 1회, 테스트용 반복 표시)                   |
| `flows[].urlPattern`           | 특정 URL 패턴에서만 가이드를 표시                                                            |
| `flows[].activationQuery`      | 시작할 Moss 가이던스 플로우를 트리거하는 단어 또는 문장                                      |
| `flows[].requiresCompleted`    | 해당 가이드를 트리거하기 위해 완료되어야 하는 다른 가이던스 플로우 id 목록                   |
| `flows[].shouldTrigger`        | 가이드를 트리거하는 URL 등의 조건                                                            |
| `flows[].startUrl`             | 가이드를 시작하는 페이지. 사용자가 다른 페이지에 있을 경우 해당 페이지로 이동할 수 있습니다. |
| `flows[].showAskButton`        | 안내 중 사용자가 질문할 수 있는 버튼을 표시                                                  |
